PeerTutor API Documentation

Currently in development stage. Check back later for an official release!

In order to use the API, use the 'curl' method in terminal or a URL Session of some kind that supports both GET and POST requests

Currently Allowed Api Requests:

Look up a user (GET) @ /api/user/#

No Parameters (GET)
Example return JSON:

{ "email": "example2@gmail.com", "frees": [ "d1p6" ], "grade": 11, "id": 2, "joinDate": "Mon, 18 Oct 2021 14:17:37 GMT", "permission": 1, "teachableGrades": "8,9,10,11,12", "username": "example2" }

Look up a course (GET) @ /api/course/#

No Parameters (GET)
Example return JSON:

{ "description": "Calc", "id": 1, "name": "Math" }

Find matches by tutee id (GET) @ /api/findmatchbytutee/#

No Parameters (GET)
Example return JSON:

{ "matches: ": [ { "course_id": 1, "id": 1, "period": "d1p6", "tutee_id": 2, "tutor_id": 1 } ] }

Find matches by tutor id (GET) @ /api/findmatchbytutor/#

No Parameters (GET)
Example return JSON:

{ "matches: ": [ { "course_id": 1, "id": 1, "period": "d1p6", "tutee_id": 2, "tutor_id": 1 } ] }

Find course requests by id (GET) @ /api/findcourserequestsbyid/#

No Parameters (GET)
Example return JSON:

{ "id": 2, "user_id": 1, "course_id": 1, "relationship": True }

Register a user (POST) @ curl http://hackleypeertutor.herokuapp.com/api/register -d "username=example2" -d "email=example2@gmail.com" -d "grade=11" -d "password=examplePassword" -d "frees=d1p2d3p5d5p6" -X POST -v

Parameters: username (String) email (String) grade (Int) password (String) frees (Undelimited String)
Example return JSON:

Request a course for a user (POST) @ curl http://hackleypeertutor.herokuapp.com/api/request -d "user_id=1" -d "course_id=2" -d "relationship=True" -X POST -v

Notes: Relationship True suggests teacher and relationship False suggests student

Parameters: user_id (Int) course_id (Int) relationship (Boolean -- True->Teacher, False->Student)
Example return JSON:
